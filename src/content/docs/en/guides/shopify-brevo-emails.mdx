## Solution Architecture

Why choose the Brevo Transactional API over integrated solutions?

The **Shopify Admin REST → Python → Brevo API v3** stack offers total flexibility. Unlike "no-code" automations that charge per execution (like Zapier), a Python script hosted on a VPS or AWS Lambda allows you to process thousands of orders for almost zero cost.

The architecture relies on a polling cycle or a trigger (Webhook) that extracts order data from Shopify to inject it into a highly optimized Brevo template.

## Step-by-Step Implementation Guide

### 1. Access Configuration
Create a custom app in your Shopify administration. You must grant the `read_orders` scope. Retrieve the **Admin API Access Token**. On the Brevo side, go to your settings to generate a **v3 API Key**.

### 2. Developing the Synchronization Engine
The following script implements production-grade logic: error handling, automatic retries for rate-limiting, and real pagination of orders.

```python title="shopify_brevo_sync.py"
import os
import requests
import time
import re
from dotenv import load_dotenv
from requests.adapters import HTTPAdapter
from urllib3.util.retry import Retry

load_dotenv()

# Environment Configuration
SHOPIFY_STORE_URL = os.getenv("SHOPIFY_STORE_URL")
SHOPIFY_ACCESS_TOKEN = os.getenv("SHOPIFY_ACCESS_TOKEN")
BREVO_API_KEY = os.getenv("BREVO_API_KEY")
BREVO_SENDER_EMAIL = os.getenv("BREVO_SENDER_EMAIL")

def get_session():
    """
    Configure a session with exponential backoff.
    Crucial to avoid 429 (Too Many Requests) blocks.
    """
    session = requests.Session()
    retry = Retry(
        total=5,
        backoff_factor=1,
        status_forcelist=[429, 500, 502, 503, 504],
        allowed_methods=["HEAD", "GET", "POST", "OPTIONS"]
    )
    adapter = HTTPAdapter(max_retries=retry)
    session.mount("https://", adapter)
    return session

def fetch_shopify_orders(session):
    """
    Retrieve orders via Shopify REST API.
    Handles pagination via 'Link' header.
    """
    orders = []
    # Targeting recent orders with a limit of 50 per page
    url = f"{SHOPIFY_STORE_URL}/admin/api/2024-04/orders.json?status=any&limit=50"
    headers = {"X-Shopify-Access-Token": SHOPIFY_ACCESS_TOKEN}
    
    try:
        while url:
            response = session.get(url, headers=headers)
            response.raise_for_status()
            data = response.json()
            orders.extend(data.get("orders", []))
            
            # Parse Link header for the next page
            link_header = response.headers.get("Link")
            if link_header:
                match = re.search(r'<(.*?)>; rel="next"', link_header)
                url = match.group(1) if match else None
            else:
                url = None
    except requests.exceptions.HTTPError as e:
        print(f"Shopify API Error: {e.response.status_code}")
        raise e
    return orders

def send_brevo_email(session, customer_email, order_name):
    """
    Send email via Brevo transactional endpoint.
    """
    url = "https://api.brevo.com/v3/smtp/email"
    headers = {
        "api-key": BREVO_API_KEY,
        "Content-Type": "application/json"
    }
    
    # Structured payload for Brevo v3
    payload = {
        "sender": {"email": BREVO_SENDER_EMAIL, "name": "My Shopify Store"},
        "to": [{"email": customer_email}],
        "subject": f"Order Confirmation {order_name}",
        "htmlContent": f"<html><body><h1>Thank you!</h1><p>Your order {order_name} has been validated.</p></body></html>"
    }
    
    try:
        response = session.post(url, json=payload, headers=headers)
        response.raise_for_status()
    except requests.exceptions.HTTPError as e:
        print(f"Brevo API Error: {e.response.status_code}")
        raise e

def main():
    session = get_session()
    try:
        orders = fetch_shopify_orders(session)
        for order in orders:
            email = order.get("email")
            name = order.get("name")
            if email:
                send_brevo_email(session, email, name)
    except Exception as e:
        print(f"Critical Failure: {e}")

if __name__ == "__main__":
    main()
```

## Error Handling and Security

### Securing API Keys
The most common mistake is "hardcoding" credentials. Always use a `.env` file added to your `.gitignore`. In production, prefer GitHub Actions secrets or AWS/GCP environment variables.

### The Rate Limiting Challenge (Error 429)
Both Shopify and Brevo limit the number of requests per second. If you process 10,000 orders at once, you will receive a `429 Too Many Requests` error. The solution implemented here is **Exponential Backoff** via `urllib3.util.retry`. The script waits for an increasing amount of time between attempts, ensuring the flow is not interrupted.

## Optimization and Scalability

To move from a dozen orders to industrial volume (10k+ / day), polling (`GET /orders.json`) becomes inefficient.

**The logical evolution: Webhooks.**
Instead of asking Shopify "Are there any new orders?", Shopify sends a `POST` notification to your server as soon as an order is created. You then replace the `main()` function with a Flask or FastAPI endpoint that receives data in real-time and triggers the Brevo call instantly.
